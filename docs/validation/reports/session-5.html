Verdict: every Verified-by assertion is met on observed evidence except CI, which cannot run until the PR exists and is recorded pending below. Both engines run as jobs from the conformance driver with streamed partials observed in deterministic order and cancellation honored; every emitted product carries authority exploratory and a kernel set hash equal to an independent frames layer's over the same bytes; the full root gate, the strict seam, and the sibling baseline compare are green.

Residual risk and honest carryover. First, the schema is v0 by design: the geometry and series kinds have payload shapes and unit carriage but no engine emitting them yet (the grammar demo of Session 6 exercises intervals and field first; series arrives with the eval-series wiring, geometry with the footprint path). Second, jobs run in-process; the worker-hosted job runner (the app's coverage worker pattern generalized) is deliberately not built until the panel work needs it, so cancellation today is cooperative-only between cells, targets, and yields, never preemptive. Third, provenance is stamped once at submit time: a job that furnishes kernels mid-run would carry a stale set hash, so ComputeEnv routes furnish through the frames layer and no adapter furnishes mid-job; a runtime guard is deferred until a host adapter makes mid-job furnishing reachable. Fourth, the existing app and SDK call paths (the bespoke coverage worker protocol, the SDK's inert AbortSignal) are not yet migrated onto JobHandle; that convergence is W3 and later work, recorded here so the three ad hoc patterns do not read as intended permanence.
