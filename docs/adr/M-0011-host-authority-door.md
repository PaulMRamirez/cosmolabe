# M-0011: The host-authority door opens through data adapters only

Status: Accepted (provisional, review-on-return)
Date: 2026-07-11 | Deciders: Paul Ramirez, solo per the mandate (docs/collab/mandate.md)
Review-on-return: yes; the first exercise of rule 4's host door and the adapter fidelity contract

## Context
M-0004 defined `authority: 'host' | 'exploratory'` and reserved the host side: engines emit 'exploratory', the job runner refuses jobs asserting authority, and the door through which 'host' enters was to be opened by ADR with the first host data adapter, not by default. Session 8 builds that first adapter (MMGIS-shaped computed layers into the panel), so the door's rules are due now, before any host product renders. The governing principle is the PlanDev source-of-truth layering of docs/design/01: authoritative quantities come from the host's own model, the panel renders them and computes only exploratory quantities beside them, and consistency is bought with shared provenance rather than cross-runtime coupling.

## Decision
Host authority enters through host data adapters and nowhere else. An adapter is a pure mapping module whose duty is fidelity, not computation: every provenance field carries the host's own identity in the host's own terms (the host tool as `engine`, the host's dataset identifier as the kernel-set identity, prefixed `host:` so it can never be mistaken for one of our kernel hashes, the host's generation timestamp as `computedAt`), and the only transformations permitted are unit and coordinate conversions documented at the adapter boundary (rule 9). An adapter that computes, corrects, resamples, or otherwise improves host data has left its charter; that work belongs to an engine and emits 'exploratory'. Engines and the job runner keep their M-0004 posture unchanged (emit 'exploratory', refuse asserted authority). The panel renders host and exploratory products side by side on one surface with the provenance chip distinguishing them; that side-by-side is the authoritative-plus-exploratory criterion, and no surface may visually promote an exploratory product to look authoritative. The first adapter is `createMmgisDataAdapter` in `@bessel/panel` (`mmgis-adapter.ts`), whose MMGIS-shaped input type is honestly a fixture shape, to be refined against the MMGIS repository when integration work is scheduled there.

## Consequences
Rule 4 now has a positive statement, not only a prohibition: reviewers check adapters for fidelity (no computation, host terms carried through) rather than hunting for authority assignments. Each new host (Aerie, live MMGIS) adds an adapter under this charter without reopening the ADR; a host whose data model cannot map faithfully triggers a schema conversation (M-0004's posture). The panel's provenance grammar gains its second word: 'Host data' chips beside 'Computed here' chips on one surface.
