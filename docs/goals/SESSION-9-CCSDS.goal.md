# Goal: the CCSDS round-trip suite (Session 9, the W4 stretch)

Session 8 (the host-panel contract, the first host data adapter, the MMGIS-shaped embed behind a flag) is complete: the sync surface is typed and exercised live, M-0011 opened rule 4's host door with the fidelity charter, and the fixture host renders host and exploratory products side by side. W4's remaining unmet criteria are the physical-device items, and those need hardware: docs/goals/SESSION-DEVICE-TRUTH.goal.md preempts this goal the moment an iPhone is in hand, and closing the W4 gate waits on that session either way. This session takes the W4 stretch, the CCSDS round-trip suite, which needs no hardware and hardens the interchange story the validation page will eventually cite.

Context: `@bessel/interop` already reads and writes OEM and AEM, reads CDM, and exports CZML, each with unit tests. What does not exist is the suite: a systematic round-trip discipline (write, re-read, compare against the source states with stated numeric tolerances, not eyeballed sameness), fidelity pinned per format and per field, and the bridge into the compute plane so an ingested ephemeris becomes an AnalysisProduct with provenance carried honestly (the file is host-shaped data: whose states these are, which originator, what units, is recorded, and the authority question is answered by M-0011's charter, an ingest adapter maps faithfully and never computes).

Steps: first, the round-trip suite proper: for OEM and AEM, generate ephemerides and attitude from the engines over the GS-2 fixtures, write, re-read, and assert states match the source at stated tolerances (positions at or below meter scale, the tolerance recorded per format in the suite, failures naming the field); include a header and metadata fidelity pass (originator, object ids, time systems, frames) since interchange dies on metadata drift, and pin the CDM read against its committed fixture. Second, the ingest bridge: an OEM file read becomes a geometry or series AnalysisProduct through an adapter under the M-0011 charter, provenance carrying the file's own originator and creation date in host terms with a file: prefixed kernel-set identity, and the CLI named as the consumer that round-trips products to files (the f64le-base64 encoding of M-0004 remains the JSON contract; CCSDS text formats are the interchange contract, and the suite states which fidelity each guarantees). Third, if capacity holds: a Horizons or SPICE-truth cross-check lane for one OEM round trip (states written from the frames tier compare against the same states recomputed after ingest), wiring the suite into the differential-harness habit of measuring rather than asserting.

Verified by: the round-trip suite green in `pnpm verify` with tolerances stated in the test names or assertions; metadata fidelity pinned; the OEM ingest adapter producing a product whose provenance carries the file's own terms, pinned by unit test; `pnpm verify` green locally and in CI; strict seam green; `TZ=America/Los_Angeles BASELINE_DIR=tests/golden/session-4-repoint node scripts/baseline.mjs compare` green; the carryover ledger updated with what this session executed and what it did not.

Exit: run scripts/review-on-return.sh and refresh the register and brief; run /verify-spec and commit the report; draft the next goal file (the device-truth session if hardware is in hand, else the W4 gate evaluation with its honest carryover of the device items).
