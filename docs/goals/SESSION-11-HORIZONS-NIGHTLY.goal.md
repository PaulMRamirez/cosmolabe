Steps: first, the spot-check script: query the JPL Horizons API (the public text or JSON interface, no key required) for a small fixed set of states drawn from the golden scenarios (Cassini from Saturn at the GS-2 SOI epoch, Earth and Mars barycenter from the Sun in the GS-4 era window), compute the same states through the frames tier over the fetched kernels, and compare with stated tolerances that respect the ephemeris difference (the fixtures are de440s; Horizons serves its own solution set, so the tolerance is kilometers-scale physical agreement, stated per body and recorded in the script, not the harness's bit-parity). Second, the nightly lane: a scheduled workflow (cron, off-peak) that runs fetch-kernels and the spot-check and fails loudly on a tolerance breach; it must also be manually dispatchable so the first observed run does not wait a day, and its failure mode distinguishes "Horizons unreachable" (a skip with a named reason, not a red X that trains alarm fatigue) from "states disagree" (the real alarm). Third, surface the result: a row or section on the validation page generated from the committed table, so the public claims page carries the external-truth lane beside the internal-parity lanes. Network reality is part of the evidence: if the Horizons API shape differs from what this goal assumes, record what was actually observed and adjust; if CI cannot reach Horizons at all, the honest outcome is the script plus a local observed run plus the workflow marked with its blocker, never a green lane that never ran.
